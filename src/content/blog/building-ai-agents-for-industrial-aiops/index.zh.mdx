## 为工业 AIOps 构建 AI Agent

工业 AIOps——将 AI 应用于 IT 和 OT 运维——与构建聊天机器人或内容生成器有着根本的不同。风险更高，数据更杂，系统必须在严格的可靠性约束下运行。

本文分享了构建 EMU Agent 的经验，这是一个面向储能 O&M（运维与维护）的智能诊断系统。

## 问题空间

储能系统非常复杂。一个典型站点包括：

- **电池架**，包含数百个单体电芯
- **功率转换系统（PCS）**，管理充放电
- **EMS（能源管理系统）**，协调电网交互
- **环境控制**（HVAC、消防）
- **通信网络**（Modbus TCP、IEC61850 GOOSE/MMS）

当出现问题时，运维人员面临告警洪流。单个根因可能触发跨子系统的级联告警。诊断实际问题需要：

1. 从告警流中过滤噪声
2. 跨子系统关联告警
3. 查询实时传感器数据获取上下文
4. 参考故障规范和历史事件
5. 综合出连贯的诊断报告和可操作建议

这正是 AI Agent 能够帮助的多步骤、知识密集型任务。

## 架构概览

EMU Agent 系统有四层：

### 第一层：数据集成

原始数据来自多个来源：

- **实时**：Modbus TCP 轮询电池电压、温度、SOC
- **事件**：IEC61850 GOOSE 消息用于保护事件
- **历史**：SQLite 数据库存储快照、告警、控制命令
- **文档**：故障规范、设备手册、维护记录

FastAPI 服务层将这些标准化为统一 API，将协议细节从 Agent 逻辑中抽象出来。

### 第二层：知识库

RAG 知识库索引三类内容：

- **故障规范**：结构化故障代码及可能原因和推荐措施
- **设备文档**：参数范围、配置指南、接线图
- **运维经验**：带工程师注释的历史事件报告

检索使用混合策略：故障代码的关键词匹配、症状描述的语义搜索、参数查询的结构化查询。
