## 诊断 Agent 的提示词工程

为诊断 Agent 编写提示词与编写聊天提示词不同。以下是四个实用技巧。

## 技巧 1：约束输出格式

让 LLM 输出结构化 JSON，包含必填字段和类型约束。示例：

\`\`\`json
{
  "diagnosis": "string",
  "confidence": "number (0-1)",
  "evidence": ["string"],
  "recommended_action": "string"
}
\`\`\`

## 技巧 2：提供领域上下文

在提示词中包含具体参数范围和故障代码。例如：

- "电池单体电压必须在 2.5V 到 3.65V 之间"
- "如果温度超过 45°C，务必包含冷却建议"

## 技巧 3：使用少样本示例

包括过去正确诊断的 2-3 个示例。这显著提升了新场景中的性能。

## 技巧 4：信心校准

务必要求模型自我评估信心。低信心的诊断应触发额外的数据检索或人工审查。
